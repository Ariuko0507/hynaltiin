# Байгууллагын Бүтэц Тохиргоо

Энэхүү баримт бичиг нь байгууллагын иерархийн бүтцийг хэрхэн тохируулах талаар зааварчилгаа агуулна.

## Бүтцийн Дүр зураг

```
Захирал 1 (Батаар)
├── Захирал 2 (Темүжин)
└── Төвийн Менежер (Хэнбиш)
    ├── Албаны Дарга 1 (Энх) → Санхүүгийн Хэлтэс
    │   └── Баг Удирдагч 1 (Ойүу)
    ├── Албаны Дарга 2 (Энх) → Хүний Нөөцийн Хэлтэс
    │   └── Баг Удирдагч 2 (Сарнай)
    ├── Албаны Дарга 3 (Энх) → Үйл ажиллагааны Хэлтэс
    │   └── Баг Удирдагч 3 (Түвшин)
    ├── Албаны Дарга 4 (Энх) → Борлуулалтын Хэлтэс
    │   └── Баг Удирдагч 4 (Нара)
    ├── Албаны Дарга 5 (Энх) → Маркетингийн Хэлтэс
    │   └── Баг Удирдагч 5 (Болд)
    └── Албаны Дарга 6 (Энх) → Технологийн Хэлтэс
        └── Баг Удирдагч 6 (Ганзориг)
```

## Тохиргооны Файлууд

### 1. SQL Тохиргоо (`organizational_structure_setup.sql`)
- PostgreSQL скрипт
- Бүх хүснэгтүүд, өгөгдлийг тохируулна
- Шууд базад ажиллуулах боломжтой

### 2. Python Тохиргоо (`scripts/setup_organizational_structure.py`)
- Supabase клиент ашигладаг
- Async/await бүтэцтэй
- Орчны хувьсагч ашигладаг

### 3. XML Өгөгдөл (`data/odoo_organizational_data.xml`)
- Odoo системд зориулсан XML формат
- Record ID-тэй тодорхойлолт

### 4. Odoo Manifest (`__manifest__.py`)
- Odoo модулийн тохиргоо
- Хамаарал болон дата файлуудыг тодорхойлно

## Хэрэгжүүлэх Алхамууд

### Алхам 1: Орчны хувьсагчдыг тохируулах
```bash
# .env файлд нэмэх
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key
```

### Алхам 2: Python скрипт ажиллуулах
```bash
pip install supabase
python scripts/setup_organizational_structure.py
```

### Алхам 3: SQL скрипт ажиллуулах (хэрэв шаардлагатай бол)
```bash
psql -h your_host -U your_user -d your_database -f organizational_structure_setup.sql
```

## Үүрэг болон Хариуцлагууд

### Захирал 1 (Батаар)
- **Гол үүрэг**: Батлах болон comment бичих
- **Даалгавар шалгах**: Харагдах бол засуулна
- **Биелэлт авах**: Батлагдсан даалгаврыг биелэлт болгон авч, засвар хэрэгтэй үед засуулах батлах хэрэгтэй

### Захирал 2 (Темүжин)
- **Даалгавар шалгах**: Зүйл байвал засуулана
- **Биелэлт шалгах**: Биелэлтийг шалгаж засах зүйл байвал шалгана

### Төвийн Менежер (Хэнбиш)
- **Хурал удирдах**: Хурлын үеээр voice хийх
- **Даалгавар бүтээх**: Даалгавар хийнэ, захирал 1-д шалгуулж засах зүйл байвал засна
- **Батлуулах**: Дараа нь захирал 2-д өгч засах зүйл байвал засж дараа нь батлуулна
- **Хуваарилах**: Баталсан даалгаврыг доод хэлтсийнхандаа хуваарилж өгнө
- **Биелэлт авах**: Даалгавраа биелэлт болгон авч нэгтгэн захирал 1-д шалгуулна

### Албаны Дарга (Энх)
- **Хурал оролцоо**: Хурлын үеээр voice хийх
- **Даалгавар менежмент**: Даалгавраа захирал 1-д шалгуулж засах зүйл байвал засна
- **Батлуулах процесс**: Дараа нь захирал 2-д өгч засах зүйл байвал засж дараа нь батлуулна
- **Доод шатны удирдлага**: Баталсан даалгаврыг доод хэлтсийнхандаа хуваарилж өгнө

### Багийн Удирдагч (Leader)
- **Даалгавар олгох**: Ажилчдад даалгавар өгч
- **Гүйцэтгэл хянах**: Даалгаврыг хийлгэж биелэлт болгон авна
- **Тайлангууд нэгтгэх**: Нэгтгээд хэлтсийн даргадаа буцааж өгнө

## Веб Функционалууд

### 1. File PDF татах
- Бүх ажилчид үүрэг болон биелэлт авах боломжтой

### 2. Цагийн тохиргоо
- Бүх ажилчдад өөрсдийн гэсэн хуваарьтай байна

### 3. Ажилчдыг үнэлэх систем
- Manager т ямар баг ямар үйл явцтай багааг харуулна
- Хугацаа хэтэрсэн үед торгох

### 4. Мэдэгдэл хүртээмж
- Нэг хэлтэст даалгавар өгөхөд бусад хэлтсүүдэд мэдэгдэл болгон очно

### 5. Хурал зарлах систем
- Manager хурал зарлах үед ямар төрлийн хурал гэдгээс сонгоно
- Төрлөөс хамаарч хүн сонгох, нийтээр гэх мэт хурал үүсгэнэ

### 6. Биелэлтийн хяналт
- Биелэлт авах хугацаа хэтэрсэн тохиолдолд биелэлт авахгүй
- Manager т өгөөгүй гэж хадгалагдана
- Диаграм дээр харагдана

## Өгөгдлийн Бүтэц

### Хүснэгтүүд:
- `users` - Хэрэглэгчид
- `departments` - Алба хэлтсүүд
- `roles` - Үүрэгүүд
- `positions` - Албан тушаал
- `hierarchy_levels` - Иерархийн түвшинүүд
- `teams` - Багууд
- `management_assignments` - Менежмент томилгоо
- `reporting_chains` - Удирдлагын суваг
- `organizational_structure` - Байгууллагын бүтэц

## Холбоосууд

- Бүх хэрэглэгчид `manager_id`-аар удирдагчаа заана
- Алба хэлтсүүд `manager_id`-аар даргаа заана
- Удирдлагын суваг нь шууд удирдагчийн холбоог хадгална
- Байгууллагын бүтэц нь бүх иерархийн мэдээллийг агуулна

## Тестлэх

Скриптийг ажиллуулсны дараа дараах query-г ажиллуулж үр дүнг шалгана уу:

```sql
SELECT 
    u.name,
    u.position,
    d.name as department,
    m.name as manager,
    hl.title as hierarchy_level
FROM public.users u
LEFT JOIN public.departments d ON u.department_id = d.id
LEFT JOIN public.users m ON u.manager_id = m.id
LEFT JOIN public.organizational_structure os ON u.id = os.user_id
LEFT JOIN public.hierarchy_levels hl ON os.position_level = hl.level_number
ORDER BY hl.level_number, d.name;
```

Энэ нь таны байгууллагын бүтцийг зөв тохируулсан эсэхийг харуулна.
